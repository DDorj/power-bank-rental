# Ажлын Даалгавар ба 60 Хоногийн Төлөвлөгөө

> **Баримтын зорилго:** Backend, mobile app, dashboard хөгжүүлэлтийн ажлын хүрээ, ерөнхий даалгавар, 60 хоногийн хугацаатай хэрэгжилтийн төлөвлөгөөг товч байдлаар тодорхойлох.

---

## 1. Ажлын ерөнхий хүрээ

Төслийн хүрээнд дараах үндсэн системүүдийг боловсруулна:

1. `Backend system`
2. `Mobile application`
3. `Admin dashboard`

Ажил нь шаардлага тодорхойлох, хөгжүүлэлт хийх, системүүдийг хооронд нь холбох, шалгах, хүлээлгэн өгөх үе шатуудтай байна.

---

## 2. Backend хөгжүүлэлтийн даалгавар

- Хэрэглэгчийн бүртгэл, нэвтрэлт, үндсэн эрхийн бүтэц боловсруулах
- Station, power bank, rental үйл ажиллагааны үндсэн API хөгжүүлэх
- Wallet болон төлбөрийн урсгалын үндсэн логик боловсруулах
- Mobile app болон dashboard-д ашиглагдах өгөгдлийн API бэлтгэх
- Notification, log, basic security тохиргоо хийх
- API баримтжуулалт, тестийн үндсэн бэлтгэл хийх

---

## 3. Mobile app хөгжүүлэлтийн даалгавар

- Аппын ерөнхий бүтэц, navigation, үндсэн дэлгэцүүдийг боловсруулах
- Нэвтрэх, бүртгүүлэх, хэрэглэгчийн үндсэн хэсгийг хийх
- Station list, map, detail дэлгэцүүдийг хөгжүүлэх
- QR scan, rental start, active rental, return flow хийх
- Wallet, payment, history, notification дэлгэцүүдийг холбох
- Туршилт, сайжруулалт, release build бэлтгэх

---

## 4. Dashboard хөгжүүлэлтийн даалгавар

- Admin нэвтрэх болон хандах эрхийн үндсэн бүтэц хийх
- User, station, power bank удирдлагын хэсэг хөгжүүлэх
- Rental, wallet, payment хяналтын хэсэг хийх
- Шүүлтүүр, жагсаалт, үндсэн тайлангийн хэсэг нэмэх
- Системийн асуудал, төлөв, лог харах хэсэг боловсруулах
- Dashboard-ийг backend API-тай бүрэн холбох

---

## 5. Deliverable

Ажлын үр дүнд дараах зүйлс хүлээлгэн өгнө:

- Backend source code
- Mobile app source code
- Dashboard source code
- Database schema болон үндсэн бүтэц
- API баримтжуулалт
- Ашиглалтын товч тайлбар
- Deployment болон handover-д хэрэгтэй үндсэн материал

---

## 6. 60 Хоногийн Timeline

### 1-р үе шат: `1–10 дахь хоног`

- Шаардлага, feature list-ийг эцэслэх
- System architecture болон өгөгдлийн бүтцийг батлах
- Backend, mobile app, dashboard ажлын ерөнхий бүтэц гаргах

### 2-р үе шат: `11–20 дахь хоног`

- Backend суурь бүтэц эхлүүлэх
- Mobile app суурь бүтэц, navigation эхлүүлэх
- Dashboard суурь бүтэц эхлүүлэх
- Auth, user, station, power bank үндсэн хэсгүүдийг боловсруулах

### 3-р үе шат: `21–35 дахь хоног`

- Rental flow-ийн үндсэн хөгжүүлэлт хийх
- Wallet болон payment integration хийх
- Mobile app-ийн station, QR, rental screen-үүдийг боловсруулах
- Dashboard дээр үндсэн удирдлагын хэсгүүдийг хийх

### 4-р үе шат: `36–45 дахь хоног`

- Return flow, history, notification хөгжүүлэх
- Backend, mobile app, dashboard хоорондын уялдааг бүрэн холбох
- Dashboard тайлан, хяналтын хэсгийг өргөтгөх
- Дотоод туршилт эхлүүлэх

### 5-р үе шат: `46–55 дахь хоног`

- Алдаа засварлах
- Урсгал, интерфэйс, гүйцэтгэлийн сайжруулалт хийх
- Demo болон хүлээлгэн өгөх хувилбар бэлтгэх

### 6-р үе шат: `56–60 дахь хоног`

- Final QA хийх
- Release / deployment бэлтгэх
- Баримтжуулалт дуусгах
- Хүлээлгэн өгөх, хаалтын шатны шалгалт хийх

---

## 7. Milestone

- `Milestone 1:` Шаардлага, бүтэц, ерөнхий төлөвлөгөө батлагдсан
- `Milestone 2:` Backend, mobile app, dashboard суурь хөгжүүлэлт бэлэн
- `Milestone 3:` Гол feature-үүд хоорондоо холбогдсон demo бэлэн
- `Milestone 4:` Финал хувилбар, баримтжуулалт, хүлээлгэн өгөлт дууссан

---

## 8. Тайлбар

Энэхүү баримт нь гэрээний хавсралт байдлаар ашиглахад зориулагдсан ерөнхий ажлын даалгавар болно. Дэлгэрэнгүй техникийн шаардлага, нэмэлт feature, гуравдагч талын интеграцийн нарийвчилсан нөхцөлийг тусдаа техникийн даалгавар эсвэл нэмэлт хавсралтаар тодорхойлж болно.
